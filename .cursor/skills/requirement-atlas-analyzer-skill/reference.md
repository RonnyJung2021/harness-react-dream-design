# Requirement Atlas Analyzer — 参考

本文件为 [SKILL.md](SKILL.md) 的补充。

## 文首 / 文末声明的最低覆盖点（检查清单）

生成或审阅 `*-requirement-atlas.md` 时确认：

- [ ] 明确写出：**仅供参考**，非完整需求分析。
- [ ] 明确写出：关系/影响可能**不全面**，可能**准但不全**。
- [ ] 明确写出：实现过程中文档会与仓库**逐渐不一致**，须**谨慎以当前代码为准**。
- [ ] 文首与文末各出现一次（文末可独立标题如「再次说明：文档性质与局限」）。

## 与实现迭代的配合

| 阶段 | 建议 |
|------|------|
| 开工前 | 生成初版文档，作为实现阶段的附加上下文 |
| 实现中 | 大改目录或契约后，更新文档或在修订记录中注明「某节已过时」 |
| 收尾 | 可选：再跑一遍本 skill 生成「对照终稿」或归档为只读历史 |

## 多需求并行

每个需求一个 `docs/requirements/<slug>-requirement-atlas.md`，避免单文件混杂导致上下文噪声与错误引用。
